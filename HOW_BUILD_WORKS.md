# How the Build and Deployment Process Works

## 🔍 Overview

You're right to ask! Here's how the entire process works:

## 📦 What Gets Pushed to GitHub

**YES, you MUST push everything:**
- ✅ `src/` folder (all your React components, pages, etc.)
- ✅ `public/` folder (static files like favicon, PDF, images)
- ✅ `package.json` (dependencies)
- ✅ `vite.config.ts` (build configuration)
- ✅ All other config files

**What gets UPLOADED to S3:**
- ✅ Only `dist/` folder (the built website)

## 🔄 The Build Process Step by Step

### 1. You Push Code to GitHub
```bash
git push origin main
```
This pushes:
- `src/` (source code)
- `public/` (static assets)
- `package.json`
- `vite.config.ts`
- Everything else

### 2. GitHub Actions Starts

#### Step 1: Checkout Code
```yaml
- name: Checkout code
  uses: actions/checkout@v4
```
- **Downloads the ENTIRE repository** from GitHub
- Includes `src/`, `public/`, `package.json`, everything
- This happens in a fresh virtual machine (GitHub Actions runner)

#### Step 2: Install Dependencies
```yaml
- name: Install dependencies
  run: npm ci
```
- Reads `package.json`
- Installs all npm packages (React, Vite, etc.)
- Creates `node_modules/` folder

#### Step 3: Build the Project
```yaml
- name: Build project
  run: npm run build
```
This runs `vite build`, which:
1. **Reads** all files from `src/` folder
2. **Processes** React components, TypeScript, CSS
3. **Bundles** everything together
4. **Copies** files from `public/` folder to `dist/`
5. **Generates** optimized files in `dist/` folder:
   - `dist/index.html` (minified HTML)
   - `dist/assets/index-[hash].js` (bundled JavaScript)
   - `dist/assets/index-[hash].css` (bundled CSS)
   - `dist/favicon.ico` (copied from `public/`)
   - `dist/HieuVTM_CV.pdf` (copied from `public/`)
   - etc.

#### Step 4: Deploy to S3
```yaml
- name: Deploy to S3
  run: aws s3 sync dist/ s3://bucket-name/
```
- **Only uploads** the `dist/` folder to S3
- S3 only contains the **built website**, not source code

## ✅ What This Means

### Must be in Git Repository:
- ✅ `src/` - Source code (needed for build)
- ✅ `public/` - Static files (needed for build)
- ✅ `package.json` - Dependencies (needed for build)
- ✅ `vite.config.ts` - Build config (needed for build)

### Should NOT be in Git Repository:
- ❌ `node_modules/` - Generated locally, installed by CI/CD
- ❌ `dist/` - Generated by build, uploaded to S3 separately

### What's on S3 (Production):
- ✅ Only `dist/` contents (built website)
- ❌ No `src/` (source code stays on GitHub)
- ❌ No `public/` (already copied into `dist/`)

## 🔒 Security Note

This is actually **secure**:
- Source code (`src/`) stays on GitHub (private repository)
- Only built files (`dist/`) are on public S3
- Users can't see your source code, only the compiled website

## 🧪 Test It Locally

To understand the build process, try this locally:

```bash
# 1. Install dependencies
npm ci

# 2. Build the project (same as GitHub Actions)
npm run build

# 3. Check what was generated
ls -la dist/

# You should see:
# - index.html
# - assets/ folder with JS/CSS
# - Files from public/ folder (favicon.ico, etc.)
```

## 📝 Summary

**Your question:** "Don't you have to push public, src? How can they generate the web without them?"

**Answer:** YES! You absolutely MUST push `src/` and `public/` to GitHub because:
1. GitHub Actions checks out the entire repo (including `src/` and `public/`)
2. The build process (`npm run build`) reads from `src/` and `public/`
3. It generates `dist/` folder
4. Only `dist/` gets uploaded to S3

**The workflow file handles this automatically** - it checks out everything, builds it, then uploads only the result!
